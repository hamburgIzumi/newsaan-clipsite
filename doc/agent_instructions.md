# エージェント指示書: Twitchクリップ閲覧サイト開発

本作戦書は、Twitch配信者 `newsaan` のクリップを閲覧・検索するWebサイトを開発するための、各役割（エージェント）への詳細な指示書です。

---

## 1. プロジェクト概要

*   **プロジェクト名**: newsaan-clipsite
*   **対象の配信者**: [newsaan (Twitch)](https://www.twitch.tv/newsaan)
*   **開発目標**: Twitchのクリップをサイト内で完結して閲覧・検索・ソートできるモダンなWebサイトの構築。
*   **公開方法**: GitHub Pagesによる静的サイト公開

---

## 2. Webサイト要件定義

### 2.1 機能要件
1.  **クリップ表示・閲覧機能**:
    *   外部遷移せず、当サイトのプレイヤー（iframe等）でクリップの再生が完結すること。
2.  **文字列検索機能**:
    *   クリップのタイトルやゲーム名などから特定のクリップを部分一致検索できること。
3.  **ソート機能**:
    *   閲覧数（Views）の多い順・少ない順での並び替え機能。
4.  **日付検索機能**:
    *   クリップの作成日時（作成日範囲）によるフィルタリング機能。
5.  **高度な検索条件**:
    *   検索ワードや日付、ソート条件を「AND（かつ）」および「OR（または）」で複合して検索できること。

### 2.2 非機能要件・デザイン
*   **デザイン**: 暗色系（ダークモード）を基調とした、現代的でモダンなゲーミングライクな雰囲気。レスポンシブ対応（スマホ・PC両対応）。
*   **ディレクトリ構造**:
    *   `/img`: 画像やアイコン（png, svg等）を配置
    *   `/js`: JavaScriptファイル（`app.js`等）を配置
    *   `/css`: スタイルシート（`style.css`等）を配置
    *   `index.html`: メインのHTMLファイル

## 4. 開発時の補足ルール
*   **タイトル、ISSUEおよびコミットメッセージ、プルリクエストの記述などは日本語を使用すること。**
*   **Gitブランチ戦略 (厳守)**:
    *   **ベース**: 常に `main` から分岐した **`work` ブランチ** を開発の主軸とします。
    *   **各Issueの開発**: 各Issue用のブランチ（例: `feat/issue-X-...`）は、`main` からではなく、**必ず `work` ブランチから分岐** して作成してください。
    *   **プルリクエスト (PR)**:
        *   各IssueのPRは、`work` ブランチ宛てに作成し、レビュー後に `work` へマージします。
        *   すべてのIssueが消化された後、**`work` ブランチから `main` ブランチへのPR** を作成します。
    *   **マージ制限 (重要)**:
        *   `work` から `main` へのPRを作成した後は、**自動マージ（merge）を行わない** でください（保留状態にし、ユーザーの指示または手動操作に委ねます）。
*   TwitchのAPIからクリップを取得する部分について、APIキーの制限や認証が必要な場合は、静的なJSONデータ（モックデータ）を作成して読み込める設計にし、ローカルでも完全に動作するように配慮すること。
*   コード内のコメントは原則として **日本語** で記述すること。

## 5. 追加要件（スマートフォン向けレスポンシブ対応）

### `[Issue #6] スマートフォン向けレスポンシブデザインの最適化`
*   **目的**: スマートフォン等の狭画面幅で閲覧した際のレイアウト崩れを修正し、PCWeb表示に一切影響を与えずにモバイルファーストでプレミアムな閲覧体験を提供すること。
*   **実装内容**:
    1.  `css/style.css` にスマホ用（`max-width: 768px` および `max-width: 480px`）のメディアクエリを追加。
    2.  `.clips-header-meta` (クリップ件数・ページ数・グリッド/リスト切り替えなどのコンテナ) をスマホ表示時は縦積み (`flex-direction: column`) または折り返し (`flex-wrap: wrap`) に最適化し、重なりやはみ出しを防ぐ。
    3.  `.pagination-container` をスマホ表示時にボタンの余白やフォントサイズを最適化し、スッキリと一列に収まるようにする。
    4.  `js/app.js` を調整し、スマホなどの極小画面 (`window.innerWidth <= 480`) ではページネーションの表示範囲 (range) を動的に狭め、ボタンのあふれを防ぐ。
    5.  リストビューにおける作成者名 (`.clip-creator-info`) や再生時間バッジなどの折り返し・省略表示を最適化する。
*   **完了条件**:
    *   幅 `375px` 〜 `768px` のスマートフォン表示で、ヘッダー、ページネーション、カードグリッド/リスト、プレイヤーが崩れることなく美しく表示されること。
    *   幅 `1024px` 以上のPC表示で、既存のデザイン・レイアウトに1%も影響がないこと。
    *   コード内の追加/修正コメントはすべて日本語であること。
